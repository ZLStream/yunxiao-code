---
name: yx-commands-summary
description: 生成云效工作项总结，支持按周/月/季度/年或自定义时间范围
---

生成指定时间范围内的工作总结（含需求、任务、缺陷），适合写周报、月报或绩效总结。用户 ID 会在首次运行时通过 token 自动获取并缓存。

**步骤：**

1. 未指定时间时，运行命令会交互式提示选择时间范围：
```bash
yx-code summary
```

2. 使用预设时间周期：
```bash
# 本周总结
yx-code summary --period week

# 本月总结
yx-code summary --period month

# 本季度总结
yx-code summary --period quarter

# 本年总结
yx-code summary --period year
```

3. 使用自定义时间范围：
```bash
yx-code summary --start 2024-01-01 --end 2024-03-31
```

4. 只统计未完成的工作项：
```bash
yx-code summary --period month --status undone
```

5. 输出 JSON 格式（AI 整理展示）：
```bash
yx-code summary --period month --json
```

**参数说明：**

| 参数 | 说明 | 可选值 | 默认值 |
|------|------|--------|--------|
| `--period` / `-p` | 时间周期 | week/month/quarter/year | 交互提示 |
| `--start` | 开始日期 | 2006-01-02 格式 | - |
| `--end` | 结束日期 | 2006-01-02 格式 | 今天 |
| `--status` / `-s` | 状态过滤 | all（默认）/ undone（排除已完成）/ 中文状态名 | all |
| `--json` | JSON 格式输出 | - | false |

**AI 生成工作总结的完整流程：**

1. 获取工作项数据：
```bash
yx-code summary --period month --json
```

2. 将 JSON 数据整理成以下结构化格式展示：

```
## 工作总结（2024-03-01 ~ 2024-03-31）

**总计：N 项**（需求 X 项 / 任务 Y 项 / 缺陷 Z 项）

### 需求（X 项）
| 编号 | 标题 | 状态 | 优先级 | 创建人 | 所在项目 | 创建日期 |
|------|------|------|--------|--------|----------|----------|
| XJHS-1898 | 太保：兼容团险网点人员 | 已上线 | 中 | 谢思超 | 芯计划书 | 2024-11-26 |

### 任务（Y 项）
| 编号 | 标题 | 状态 | 优先级 | 创建人 | 所在项目 | 创建日期 |
|------|------|------|--------|--------|----------|----------|
...

### 缺陷（Z 项）
...

### 按项目汇总
| 项目 | 工作项数 |
|------|----------|
| 展业赋能 | 7 |
| 芯计划书 | 5 |
```

规则：
- 按 `category` 字段分组：`Req`=需求、`Task`=任务、`Bug`=缺陷，无数据的类别跳过
- 标题超过 30 字时截断并加"..."
- 若 `parent_id` 非空，在标题后附注"（子项）"
- 优先级为空时显示"-"
- 最后附上按项目汇总（使用 `by_project` 字段）
- 如需进一步分析，可请求："根据以上数据，帮我写一份适合汇报给领导的月度总结"
